调度原理 · 模式说明
基于 cpuset 核心分配与 sched_pixel 响应时间调节升频，不写 scaling_max_freq。本模块提供省电、均衡、系统默认三档（系统默认回写内核只读节点 response_time_ms_nom 恢复出厂升频节奏，并还原出厂 cpuset 与 uclamp.min cap=1024）；自动模式以均衡为日常底座，温度持续偏高时收口至省电，回落后恢复。需要游戏级线程调度时，在「调度接管」切换至 UGT / fas-rs / 外部调度。

后台功耗控制
后台应用限制
按包选择后台策略。默认仅预置抖音，休眠策略会在锁屏或离开前台后延时停止，前台使用不受影响。
使用说明
将需要治理的应用包名加入列表后，可按包选择限制层级。休眠会在目标 App 锁屏或离开前台后等待 3/5/10 分钟再 force-stop，用于清理短视频、广告 SDK、直播与电商尾进程，抖音通常可释放 1.5-2GB 工作集和 SDK 尾巴。前台打开时系统会临时提升，正常使用不受影响。移除包名或关闭功能会按接管前 bucket/AppOps 恢复。

待机隔离模式
仅用于 A/B 排障。开启后息屏阶段暂停模块所有待机干预（NR / SIM2 / 自动调度 / 功耗采样），验证是否为模块阻碍 deep sleep。
